# Hermetic Build Support for Konflux

This directory contains the files needed for hermetic builds in Konflux.

## Files

- **hermeto-input.json**: hermeto/cachi2 configuration for dependency prefetch
- **requirements.txt**: Python dependencies (veritas)
- **rpms.in.yaml**: Required RPM specification
- **rpms.lock.yaml**: RPM lockfile with checksums (generated by rpm-lockfile-prototype)
- **.gitmodules**: Git submodules pointing to downstream (openshift/) repos

## Submodules

Source dependencies are git submodules pointing to the downstream (openshift/) repos:

| Component | Repo | Branch |
|-----------|------|--------|
| guest-components | openshift/confidential-containers-guest-components | osc-release |
| trustee | openshift/trustee | main |
| kata-containers | openshift/kata-containers | osc-release |

## Local Build (Non-Hermetic)

```bash
podman build \
  --secret id=org,src=./org.txt \
  --secret id=key,src=./activation-key.txt \
  -f Containerfile.ubi .
```

The Containerfile detects the presence of secrets and automatically registers with subscription-manager when they exist.

## Konflux Build (Hermetic)

In Konflux, the build is **fully automatic**:

1. Cachi2 prefetches all dependencies (RPMs, cargo crates, pip packages) based on hermeto-input.json
2. Every `RUN` line is automatically prepended with `. /cachi2/cachi2.env &&` to use cached deps
3. No secrets are present, so subscription-manager registration is skipped
4. Build runs with `--network=none` (hermetic)

**The Containerfile requires ZERO changes** - Konflux handles everything transparently.
